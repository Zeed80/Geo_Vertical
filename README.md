# GeoVertical Analyzer

GeoVertical Analyzer — настольное приложение для геодезического контроля антенно-мачтовых сооружений: от импорта измерений и 3D-редактирования точек до расчета вертикальности и прямолинейности, подготовки PDF/Excel/DOCX-отчетов, шаблонов полного обследования и сохранения проектов `.gvproj`.

Актуальная пакетная версия: `2.0.0`.

## Что умеет программа сейчас

- Загружает данные из `CSV/TXT`, `DXF`, `GeoJSON`, `Shapefile`, `Trimble JXL/JobXML` и `Trimble JOB`.
- Поддерживает прямой импорт `Trimble JOB` с диагностикой качества и автоматическим использованием парного `JXL/JobXML`, если он лежит рядом.
- Объединяет измерения с другой точки стояния и помогает разруливать дубли, служебные точки и новые пояса.
- Строит и редактирует башню прямо в приложении через встроенный конструктор и 3D-предпросмотр.
- Автоматически фильтрует точки башни, группирует их по поясам и синхронизирует 3D-редактор с таблицей.
- Считает вертикальность и прямолинейность с отображением графиков, таблиц и сводных результатов.
- Формирует PDF- и Excel-отчеты по результатам расчета.
- Формирует полный DOCX-отчет по структуре `DO_TSS_2` через вкладку `Полный отчет`.
- Работает с шаблонами полного отчета и справочниками в `user_data/report_templates`.
- Сохраняет проекты в `.gvproj`, восстанавливает состояние полного отчета и поддерживает автосохранение.
- Поставляется с примерами данных в `examples/` и автоматическими тестами в `tests/`.

## Поддерживаемые форматы и Trimble

### Матрица форматов

| Формат | Статус | Примечание |
| --- | --- | --- |
| `CSV`, `TXT` | Полная поддержка | Координатные списки и табличные измерения |
| `DXF` | Полная поддержка | Импорт точек из CAD-сценариев |
| `GeoJSON` | Полная поддержка | Геоданные в JSON-представлении |
| `Shapefile (.shp)` | Поддерживается | Может потребовать дополнительные GIS-зависимости |
| `Trimble .jxl/.jobxml` | Полная поддержка | Рекомендуемый точный формат экспорта |
| `Trimble .job` | Поддерживается | Прямой импорт с диагностикой и fallback-логикой |

### Как работает импорт Trimble JOB

GeoVertical Analyzer больше не требует обязательного предварительного экспорта `JOB` в `JobXML`, чтобы открыть файл.

Текущая модель импорта такая:

1. Приложение умеет открывать `Trimble .job` напрямую.
2. Если рядом найден парный `JXL/JobXML`, приложение использует его для точного импорта и фиксирует это в диагностике.
3. Если парного экспорта нет, используется встроенный эвристический парсер `JOB`.
4. Если результат получен только из `JOB`, приложение оставляет предупреждение о желательности сверки с экспортом `JobXML`.

Практический вывод:

- для повседневной загрузки `JOB` можно открывать напрямую;
- для критичных, договорных и отчетных сценариев экспорт через Trimble Business Center по-прежнему рекомендуется как способ верификации исходных данных;
- лучший результат достигается, когда рядом с `JOB` лежит парный `JXL/JobXML`.

### Что приложение извлекает из импорта

- рабочие точки башни;
- точки стояния;
- служебные и вспомогательные точки;
- диагностические сведения о стратегии парсинга;
- сведения о парных экспортных файлах и качестве совпадения.

## Типовой рабочий процесс

### 1. Загрузка данных или создание башни

Вы можете начать двумя путями:

- загрузить измерения из файла;
- создать башню с нуля через `Файл → Создать башню...`.

Встроенный конструктор подходит для моделирования типовой башни, задания секций, поясов, размеров, формы и точки стояния без внешнего файла.

### 2. Импорт второй станции

Если измерения выполнены с нескольких стояний:

- загрузите базовый набор данных;
- используйте `Импорт с другой точки стояния...`;
- просмотрите объединение, дубли, контрольные точки и новые пояса;
- сохраните объединенный результат для дальнейшего расчета.

Этот сценарий особенно полезен для многостанционных Trimble-измерений.

### 3. Фильтрация и редактирование

После загрузки можно:

- отфильтровать рабочие точки башни;
- проверить и скорректировать пояса;
- отредактировать координаты и служебные флаги в таблице;
- сверить результат в 3D-редакторе.

### 4. Расчет

После подготовки данных приложение:

- группирует точки по поясам;
- определяет центры и геометрию;
- считает вертикальность;
- считает прямолинейность;
- формирует таблицы, графики и сводные статусы по нормативам.

### 5. Анализ результатов

После расчета доступны:

- график вертикальности;
- график прямолинейности;
- табличные результаты по поясам/секциям;
- сводка по допускам;
- данные для экспорта в отчет.

### 6. Экспорт отчетов

Из одного проекта можно получить:

- PDF-отчет для финального представления;
- Excel-таблицы для дальнейшей обработки;
- полный DOCX-отчет по форме `DO_TSS_2`.

### 7. Сохранение проекта

Используйте `.gvproj`, если нужно сохранить:

- исходные и обработанные данные;
- состояние конструктора башни;
- результаты расчета;
- состояние вкладки `Полный отчет`;
- промежуточную работу между сессиями.

Приложение также поддерживает автосохранение и восстановление проекта после сбоя.

## Отчеты и шаблоны

### PDF

Подходит для итоговой выдачи результата расчета:

- титульная информация;
- нормативная база;
- таблицы результатов;
- графики вертикальности и прямолинейности;
- выводы и рекомендации.

### Excel

Подходит для последующей обработки и архивирования:

- исходные данные;
- центры и расчеты;
- табличная проверка допусков;
- удобный экспорт для внешней аналитики.

### DOCX: полный отчет по `DO_TSS_2`

Вкладка `Полный отчет` предназначена для подготовки полного технического отчета в формате Word.

На уровне README важно следующее:

- отчет строится по структуре `DO_TSS_2`;
- поддерживаются паспорт объекта, разделы обследования, измерительные протоколы, выводы, приложения и связанные блоки;
- расчетные данные могут подставляться автоматически;
- состояние формы сохраняется в проект `.gvproj`;
- пользователь может работать с шаблонами и переиспользовать типовые заготовки.

### Шаблоны полного отчета

Шаблоны и справочники лежат в каталоге:

```text
user_data/report_templates
```

Поддерживаются:

- сохранение и загрузка шаблонов;
- автозаполнение из расчетов;
- повторное использование данных по специалистам и приборам;
- накопление библиотек справочной информации для следующего отчета.

### Когда использовать какой формат

- `PDF` — когда нужен финальный читаемый отчет с графиками.
- `Excel` — когда нужны таблицы и дальнейшая обработка.
- `DOCX` — когда нужен полный формализованный документ обследования по структуре `DO_TSS_2`.

Подробности по содержимому и структуре профессионального отчета описаны в [docs/REPORT_FORMAT.md](docs/REPORT_FORMAT.md).

## Нормативная база

GeoVertical Analyzer не ограничивается одной исторической методикой и опирается на актуальную нормативную модель приложения.

### Ключевые нормативы для основного расчета

| Направление | Норматив | Что фиксирует приложение |
| --- | --- | --- |
| Вертикальность | `СП 70.13330.2012` | Контроль отклонения от вертикали |
| Прямолинейность | `ГОСТ Р 71949-2025` | Контроль стрелы прогиба с допуском `L / 750` |

### Нормативы для полного обследования и отчетных форм

В расширенных сценариях полного отчета и обследования также используются:

- `ГОСТ 31937-2011`;
- профильные строительные и обследовательские документы из текущей нормативной модели приложения;
- связанная нормативная база, которая подставляется в отчетные формы по типу сооружения.

### Исторический контекст

Инструкция Минсвязи СССР 1980 может упоминаться как legacy-контекст и историческая база методики, но в README она больше не позиционируется как единственный или основной текущий норматив прямолинейности продукта.

## Поддержка

Если что-то пошло не так:

1. Проверьте, что у вас Python `>= 3.10`.
2. Запустите `python -m pytest tests`, чтобы убедиться, что локальная установка рабочая.
3. Сверьте формат входного файла и поддержку зависимостей для вашего сценария.
4. Для `Trimble JOB` ориентируйтесь на диагностику импорта и, при необходимости, держите рядом парный `JXL/JobXML`.
5. Для глубоких пользовательских сценариев откройте документацию из `docs/`.
