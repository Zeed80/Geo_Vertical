# GeoVertical Analyzer

Программа для геодезического контроля антенно-мачтовых сооружений с автоматизированным расчетом отклонений от вертикали и прямолинейности согласно нормативам **СП 70.13330.2012** и **Инструкции Минсвязи СССР 1980**.

---

## 🚀 Быстрая установка

### Windows (РЕКОМЕНДУЕТСЯ)

```cmd
install_windows_simple.bat
run.bat
```

**Готово!** Программа установлена и запущена.

### Linux/macOS

```bash
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
python main.py
```

## ✨ Возможности

- **Поддержка множества форматов геоданных**: CSV, Shapefile, GeoJSON, DXF, **Trimble (JobXML, CSV, TXT)**
- **3D редактор точек с PyQtGraph**: Полнофункциональное редактирование точек в 3D (добавление, удаление, перемещение)
- **Управление поясами**: Визуальное назначение поясов точкам с цветовой кодировкой
- **Автоматическая фильтрация**: Интеллектуальное выделение точек башни из сырых данных
- **Синхронизация 3D ↔ Таблица**: Изменения в одном виджете автоматически отражаются в другом
- **Автоматическая группировка точек** по поясам с настраиваемым допуском
- **Расчет отклонений от вертикали** (d ≤ 0.001 × h)
- **Расчет стрелы прогиба** для проверки прямолинейности (δ ≤ L/750)
- **Работа с различными системами координат** (EPSG)
- **Интерактивная таблица** для редактирования данных
- **Визуализация результатов** с цветовой индикацией
- **Генерация отчетов** в форматах Excel, PDF и Word (DOCX)

- **ОС**: Windows 10/11, Linux, macOS
- **Python**: 3.8 или выше
- **ОЗУ**: минимум 2 GB (рекомендуется 4 GB)
- **Место на диске**: 500 MB (базовая установка), до 1 GB (с GDAL)

### Установка GDAL (опционально)

GDAL требуется только для работы с Shapefile. Без него программа работает с CSV, GeoJSON, DXF и Trimble форматами.

#### Windows

1. Скачайте wheel файл с https://www.lfd.uci.edu/~gohlke/pythonlibs/#gdal
2. Выберите файл для вашей версии Python (например, `GDAL-3.4.3-cp310-cp310-win_amd64.whl`)
3. Установите:
   ```cmd
   pip install GDAL-3.4.3-cp310-cp310-win_amd64.whl
   pip install geopandas
   ```

#### Альтернатива: Conda (рекомендуется для GDAL)

```cmd
conda create -n geovertical python=3.11
conda activate geovertical
conda install -c conda-forge gdal geopandas
pip install -r requirements.txt
```

## 📖 Использование

### 1. Загрузка данных

- Нажмите **"📂 Открыть"** или используйте меню **Файл → Открыть**
- Выберите файл с координатами точек:
  - **CSV, TXT** - текстовые координатные файлы
  - **Shapefile (SHP)** - геопространственные данные (требует GDAL)
  - **GeoJSON** - JSON формат для геоданных
  - **DXF** - чертежи AutoCAD
  - **Trimble** - JobXML (.jxl), CSV или TXT экспорты

**Примечание для пользователей Trimble**: Бинарные `.job` файлы требуют предварительного экспорта в JobXML через Trimble Business Center. См. [docs/TRIMBLE_SUPPORT.md](docs/TRIMBLE_SUPPORT.md) для подробностей.

### 2. Фильтрация точек (опционально)

Если файл содержит не только точки башни (реперы, точки стояния и т.п.):

1. Нажмите **"🔍 Фильтровать точки"**
2. Откроется 3D редактор с интерактивной визуализацией
3. Нажмите **"🔍 Автоматическая фильтрация"**
4. Программа автоматически:
   - Группирует точки по высоте
   - Анализирует геометрию (точки образуют окружность?)
   - Определяет ось башни
   - Исключает выбросы (реперы, оборудование)
5. Результат показывается в 3D:
   - 🟢 Зеленые = Точки башни (выбрано)
   - 🔴 Красные = Отклонено
6. При необходимости скорректируйте вручную
7. Нажмите **"✓ Применить фильтр"**

**Детальная документация:** [docs/POINT_FILTERING_GUIDE.md](docs/POINT_FILTERING_GUIDE.md)

### 3. Выбор системы координат

- В панели инструментов выберите систему координат (EPSG)
- При выборе "Авто" программа попытается определить систему автоматически
- Для расчетов данные автоматически преобразуются в метрическую систему

### 4. Редактирование данных

- На вкладке **"Данные"** можно добавлять, удалять и редактировать точки
- Используйте кнопку **"Проверить данные"** для валидации
- Некорректные значения подсвечиваются красным

### 5. Настройка параметров

- Откройте **Расчет → Параметры расчета**
- Настройте:
  - Допуск группировки по высоте
  - Метод расчета центра пояса
  - Параметры отображения графиков

### 6. Выполнение расчета

- Нажмите **"⚙️ Рассчитать"** (или F5)
- Программа:
  - Группирует точки по поясам
  - Находит центры поясов
  - Строит ось мачты
  - Вычисляет отклонения
  - Проверяет соответствие нормативам

### 7. Просмотр результатов

**Вкладка "Вертикальность":**
- График отклонений от вертикали
- Линии допуска ±0.001·h
- Цветовая индикация (зеленый - норма, красный - превышение)

**Вкладка "Прямолинейность":**
- График стрелы прогиба
- Линия допуска L/750
- Базовая линия прямолинейности

### 8. Сохранение отчета

- Нажмите **"💾 Сохранить отчет"** или Ctrl+S
- Выберите формат:
  - **PDF** - профессиональный отчет с графиками (рекомендуется)
  - **Word (DOCX)** - редактируемый отчет
  - **Excel** - табличные данные для обработки

**PDF отчет включает:**
- Титульный лист с полной информацией
- Нормативную базу (СП 70.13330.2012, Инструкция 1980)
- Исходные данные в табличном виде
- Результаты расчетов с цветовой индикацией
- Графики вертикальности и прямолинейности (высокое качество)
- Выводы и рекомендации по результатам контроля

## 📄 Формат входных данных

### CSV файлы

Минимальный формат:
```csv
X,Y,Z
10.0,0.0,5.0
7.07,7.07,5.0
...
```

Возможные названия колонок:
- X: x, X, lon, longitude, east, easting
- Y: y, Y, lat, latitude, north, northing
- Z: z, Z, h, height, elevation, alt, altitude

### Другие форматы

- **Shapefile**: Точечный слой с атрибутом высоты
- **GeoJSON**: Point геометрия с координатой Z или атрибутом высоты
- **DXF**: Объекты типа POINT

### Trimble форматы

**Поддерживаемые форматы:**
- **JobXML** (*.jxl, *.jobxml) - рекомендуемый формат для экспорта из Trimble Business Center
- **CSV** - координатные списки, экспортированные из Trimble
- **TXT** - текстовые координатные файлы

**Для работы с бинарными JOB файлами:**
1. Откройте файл в Trimble Business Center
2. Экспортируйте: `File → Export → JobXML`
3. Загрузите полученный файл в программу

**Подробная документация:** [docs/TRIMBLE_SUPPORT.md](docs/TRIMBLE_SUPPORT.md)

## 📐 Нормативная база

### СП 70.13330.2012

**Отклонение от вертикали:**
```
d_i ≤ 0.001 × h_i
```
где:
- d_i - горизонтальное отклонение центра пояса от оси (м)
- h_i - высота пояса от основания (м)

### Инструкция Минсвязи СССР (1980)

**Стрела прогиба (прямолинейность):**
```
δ_i ≤ L / 750
```
где:
- δ_i - отклонение от базовой линии прямолинейности (м)
- L - длина секции между опорными точками (м)

## 🐛 Решение проблем

### Python не найден

**Решение:**
1. Установите Python с https://www.python.org/downloads/
2. При установке отметьте "Add Python to PATH"
3. Перезапустите командную строку

### Ошибка установки PyQt6

```bash
pip install --upgrade pip setuptools wheel
pip install PyQt6 PyQt6-WebEngine
```

### Ошибка "Microsoft Visual C++ 14.0 is required"

Установите Microsoft C++ Build Tools:
https://visualstudio.microsoft.com/visual-cpp-build-tools/

### GDAL не устанавливается

1. Используйте программу без поддержки Shapefile (работает с CSV, DXF, GeoJSON)
2. Или используйте Conda для установки GDAL (см. раздел "Установка GDAL")
3. Или скачайте wheel файл вручную (см. раздел "Установка GDAL")

### Проблемы с matplotlib на Linux

```bash
sudo apt-get install python3-tk
pip install matplotlib
```
