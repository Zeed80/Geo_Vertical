# Поддержка форматов Trimble

## Обзор

Программа GeoVertical Analyzer поддерживает работу с данными от приборов Trimble. Так как формат JOB является проприетарным и недокументированным, требуется предварительный экспорт данных в поддерживаемый формат.

## Поддерживаемые форматы

### ✅ Полностью поддерживаемые

1. **JobXML (*.jxl, *.jobxml, *.xml)**
   - Текстовый XML формат от Trimble
   - Содержит полную информацию о точках и измерениях
   - Рекомендуемый формат для экспорта

2. **CSV экспорт (*.csv)**
   - Координатные списки в формате CSV
   - Простой и универсальный формат

3. **Текстовые координатные файлы (*.txt)**
   - Простые текстовые списки координат
   - Форматы: `ID N E Z` или `N E Z`

4. **Shapefile (*.shp)**
   - Стандартный геопространственный формат
   - Поддерживает систему координат (CRS)

### ⚠️ Экспериментальная поддержка

**Бинарные JOB файлы (*.job)**
- Поддерживается эвристический парсинг (точность ~90-95%)
- Проприетарный формат, рекомендуется экспорт через Trimble Business Center
- **Для важных данных**: экспортируйте в JobXML для 100% точности

## Инструкция по экспорту данных

### Вариант 1: Экспорт в JobXML (Рекомендуется)

1. Откройте ваш JOB файл в **Trimble Business Center**
2. Выберите меню `File → Export → JobXML`
3. Сохраните файл с расширением `.jxl` или `.jobxml`
4. Откройте полученный файл в GeoVertical Analyzer

**Преимущества:**
- Сохраняет всю информацию о точках
- Поддерживает метаданные и атрибуты
- Текстовый формат (можно просмотреть в текстовом редакторе)

### Вариант 2: Экспорт в CSV

1. Откройте ваш JOB файл в **Trimble Business Center**
2. Выберите меню `File → Export → CSV` или `Text File`
3. В настройках экспорта выберите:
   - ☑ Point Name
   - ☑ Northing
   - ☑ Easting
   - ☑ Elevation
4. Сохраните файл
5. Откройте полученный CSV в GeoVertical Analyzer

**Преимущества:**
- Простой универсальный формат
- Можно открыть в Excel для проверки
- Малый размер файла

### Вариант 3: Экспорт в Shapefile

1. Откройте ваш JOB файл в **Trimble Business Center**
2. Выберите меню `File → Export → Shapefile`
3. Сохраните файл
4. Откройте полученный SHP в GeoVertical Analyzer

**Преимущества:**
- Сохраняет систему координат (EPSG)
- Стандартный GIS формат
- Поддержка атрибутов точек

## Альтернативное ПО для экспорта

Если у вас нет Trimble Business Center, можно использовать:

1. **Trimble Access** (полевое ПО)
   - Export → Text File
   - Export → CSV

2. **Trimble Data Transfer** (бесплатная утилита)
   - Позволяет экспортировать JOB в текстовые форматы

## Структура данных

### Требуемые колонки для CSV

Программа автоматически определяет колонки с координатами. Поддерживаются различные названия:

- **X (Easting)**: `x`, `east`, `easting`, `lon`, `longitude`
- **Y (Northing)**: `y`, `north`, `northing`, `lat`, `latitude`
- **Z (Elevation)**: `z`, `elevation`, `elev`, `height`, `alt`, `altitude`

### Пример CSV файла

```csv
Point Name,Northing,Easting,Elevation
PT1,5500123.456,300456.789,125.340
PT2,5500234.567,300567.890,125.450
PT3,5500345.678,300678.901,125.560
```

### Пример текстового файла

```
PT1 5500123.456 300456.789 125.340
PT2 5500234.567 300567.890 125.450
PT3 5500345.678 300678.901 125.560
```

## Система координат

### Определение EPSG

1. Для **JobXML** и **Shapefile** - система координат определяется автоматически из метаданных файла

2. Для **CSV** и **текстовых файлов** - необходимо указать систему координат вручную в программе:
   - Выберите меню `Настройки → Система координат`
   - Введите код EPSG (например, 32637 для UTM Zone 37N)
   - Или выберите из списка распространенных систем

### Распространенные системы координат в России

- **EPSG:32637** - WGS 84 / UTM zone 37N
- **EPSG:32638** - WGS 84 / UTM zone 38N
- **EPSG:32639** - WGS 84 / UTM zone 39N
- **EPSG:32640** - WGS 84 / UTM zone 40N
- **EPSG:32641** - WGS 84 / UTM zone 41N
- **EPSG:32642** - WGS 84 / UTM zone 42N

## Решение проблем

### Ошибка: "Прямая загрузка бинарных JOB файлов не поддерживается"

**Причина**: Файл JOB имеет проприетарный бинарный формат

**Решение**:
1. Экспортируйте JOB в JobXML через Trimble Business Center
2. Или экспортируйте в CSV/Shapefile
3. Откройте экспортированный файл в программе

### Ошибка: "Не удалось распознать формат"

**Причина**: Неизвестная структура файла

**Решение**:
1. Убедитесь, что файл содержит колонки с координатами
2. Проверьте, что данные в правильном формате (числа, не текст)
3. Откройте файл в текстовом редакторе и проверьте структуру

### Координаты выглядят неправильно

**Причина**: Возможно, перепутаны X и Y (Easting и Northing)

**Решение**:
1. Откройте экспортированный файл в текстовом редакторе
2. Проверьте порядок колонок
3. При необходимости переименуйте заголовки колонок
