# Руководство по фильтрации точек и 3D редактору

## Назначение

Модуль фильтрации точек предназначен для автоматического выделения точек башни из общего облака геодезических измерений, которое может содержать:
- Точки поясов башни (основные данные)
- Реперы (опорные пункты)
- Точки стояния приборов
- Точки установленного оборудования
- Другие служебные точки

## Как это работает

### Алгоритм автоматической фильтрации

```
1. Группировка по высоте
   └─> Находим потенциальные пояса башни (точки на одной высоте)

2. Геометрический анализ каждой группы
   ├─> Количество точек (3-12 на пояс)
   ├─> Точки образуют окружность/многоугольник?
   ├─> Расстояния между точками примерно равные?
   └─> "Круглость" пояса (коэффициент >= 0.7)

3. Определение оси башни
   └─> Центры поясов должны быть на одной вертикальной линии

4. Исключение выбросов
   └─> Точки/группы, не вписывающиеся в геометрию башни
```

### Критерии фильтрации

**Пояс башни определяется, если:**
- ✅ 3-12 точек на примерно одной высоте (±0.1м)
- ✅ Точки образуют окружность или многоугольник
- ✅ Радиусы от центра примерно равны (круглость >= 0.7)
- ✅ Центр пояса близок к оси башни (отклонение < 1.0м)

**Точка исключается, если:**
- ❌ Одна-две точки на высоте (вероятно репер)
- ❌ Точки не образуют окружность
- ❌ Группа далеко от оси других поясов
- ❌ Слишком много точек (> 12)

## Использование

### Шаг 1: Загрузка данных

1. Загрузите файл с данными (`Файл → Открыть`)
2. В программу загружаются **все** точки из файла
3. Кнопка `🔍 Фильтровать точки` становится активной

### Шаг 3: Автоматическая фильтрация

1. **Настройте параметры** (опционально):
   - `Допуск по высоте`: 0.1м (точки в пределах ±0.1м считаются одним поясом)
   - `Мин. точек на пояс`: 3 (минимум точек для распознавания пояса)
   - `Макс. точек на пояс`: 12 (максимум точек на поясе)
   - `Порог круглости`: 0.7 (1.0 = идеальная окружность)

2. **Нажмите `🔍 Автоматическая фильтрация`**

3. **Анализ выполняется:**
   - Программа анализирует геометрию всех точек
   - Находит пояса башни
   - Определяет ось башни
   - Исключает выбросы (реперы, оборудование и т.п.)

4. **Результат отображается в 3D:**
   - 🟢 **Зеленые точки** = Точки башни (выбрано для анализа)
   - 🔴 **Красные точки** = Отклонено (реперы, оборудование)
   - Черная пунктирная линия = Ось башни

5. **Читайте детали в правой панели:**

### Шаг 4: Ручная корректировка (опционально)

Если автоматическая фильтрация ошиблась:

**Инверсия выбора:**
- Нажмите `⇄ Инвертировать` - зеленые↔красные

**Полный сброс:**
- Нажмите `↺ Сбросить` - все точки станут не классифицированными

**Изменение параметров:**
- Измените параметры фильтрации в правой панели
- Нажмите `🔍 Автоматическая фильтрация` снова

## Советы и рекомендации

### ✅ Когда автофильтрация работает лучше всего

1. **Регулярная геометрия башни**
   - Точки равномерно распределены по поясам
   - Секции примерно одинакового размера

2. **Хорошее качество съемки**
   - Минимальные ошибки измерений
   - Высоты поясов отличаются достаточно (>0.2м)

3. **Четкое разделение**
   - Реперы на земле (z≈0)
   - Точки башни существенно выше

### ⚠️ Когда может потребоваться ручная корректировка

1. **Нерегулярная геометрия**
   - Переменное сечение башни
   - Нестандартные конструкции

2. **Много измерений на одном поясе**
   - > 12 точек на поясе
   - Увеличьте `Макс. точек` параметр

3. **Близкие по высоте измерения**
   - Репер близко к поясу башни
   - Увеличьте `Допуск по высоте` или исключите вручную

### 🎯 Оптимальные параметры для разных типов башен

## Технические детали

### Алгоритмы

**Кластеризация по высоте:** DBSCAN
- Автоматически находит группы точек на одной высоте
- Устойчив к выбросам

**Геометрический анализ:** Статистический
- Центр масс точек
- Радиальное распределение
- Коэффициент вариации радиусов

**Определение оси:** Линейная регрессия
- Проходит через центры валидных поясов
- Минимизирует отклонения методом наименьших квадратов

### Метрики качества

**Circularity (круглость):**
```
circularity = 1 - (σ_radius / μ_radius)

где:
  σ_radius = стандартное отклонение радиусов
  μ_radius = средний радиус

1.0 = идеальная окружность
0.0 = точки в линию
```

**Axis Deviation (отклонение от оси):**
```
deviation = ||center_belt - axis_center||

где:
  center_belt = центр пояса (x, y)
  axis_center = проекция на ось башни
```

## Troubleshooting

### Проблема: Автофильтрация не находит пояса

**Причины:**
- Слишком строгие параметры
- Нестандартная геометрия

**Решение:**
1. Уменьшите `Порог круглости` до 0.5-0.6
2. Увеличьте `Допуск по высоте` до 0.2м
3. Измените `Мин/Макс точек` под вашу геометрию

### Проблема: Реперы определяются как пояса

**Причины:**
- Несколько реперов на одной высоте
- Случайная "круглая" конфигурация

**Решение:**
1. Увеличьте `Мин. точек` (реперов обычно 1-2)
2. Используйте инверсию `⇄` и исключите вручную

### Проблема: Часть поясов исключена

**Причины:**
- Пояс далеко от оси других поясов
- Недостаточно точек измерено

**Решение:**
1. Проверьте правильность съемки
2. Уменьшите порог `Отклонение от оси` (в коде: `axis_deviation_threshold`)

## FAQ

**Q: Можно ли сохранить настройки фильтрации?**
A: В текущей версии - нет. Запланировано в следующей версии.

**Q: Работает ли фильтрация для вытяжных труб?**
A: Да, алгоритм универсален для любых вертикальных сооружений с круглым/многоугольным сечением.

**Q: Можно ли фильтровать точки не башни, а наклонной мачты?**
A: В текущей версии - только вертикальные сооружения. Для наклонных нужна доработка.

**Q: Сколько времени занимает фильтрация?**
A: Обычно < 1 секунды для 100-500 точек.
