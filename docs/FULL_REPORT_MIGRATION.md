# Переход на новую версию полного отчёта

Новые версии GeoVertical включают расширенные структуры данных, дополнительные таблицы вкладки `📘 Полный отчёт` и переработанный DOCX‐генератор. Ниже приведён план обновления рабочих мест и действующих проектов.

## 1. Подготовка окружения

1. Обновите репозиторий до последнего коммита и установите зависимости в виртуальном окружении (`venv\Scripts\python.exe -m pip install -r requirements.txt`).
2. Перед запуском сохраните резервные копии каталога `user_data` (шаблоны, справочники) и проектов `.gvproj`.
3. Запуск приложения выполняйте в штатном режиме: `venv\Scripts\python.exe -u main.py`.

## 2. Миграция шаблонов

1. В диалоге «Шаблоны полного отчёта» откройте существующие JSON и убедитесь, что добавлены блоки:
   - `object_list` (раздел 8),
   - `documents_review` (раздел 13),
   - `technical_state` (раздел 19),
   - `conclusions` (раздел 20).
2. При отсутствии полей воспользуйтесь новым эталонным шаблоном `user_data/report_templates/DO_TSS_full.json` или сохраните проект с автоматически заполненной вкладкой `📘 Полный отчёт` и отредактируйте JSON вручную.
3. После проверки шаблонов перезагрузите выпадающий список через кнопку «🔄» на вкладке.

## 3. Обновление проектов `.gvproj`

1. Откройте каждый актуальный проект и перейдите на вкладку `📘 Полный отчёт`.
2. Нажмите «Автозаполнение» — система перенесёт результаты расчётов и сформирует новые таблицы (перечень объектов, анализ документации, оценку состояния, выводы).
3. Проверьте и дополните новые разделы вручную, затем сохраните проект, чтобы сериализовать обновлённое состояние.

## 4. Тестирование и контроль качества

1. Сформируйте DOCX для каждого ключевого шаблона и убедитесь, что документ содержит разделы 1–21 и приложения A–M. Обратите внимание на титульный лист, оглавление и колонтитулы.
2. Запустите тесты `pytest tests/test_full_report_pipeline.py` — они проверяют сериализацию расширенных структур и автозаполнение.
3. При расхождениях изучите `docs/USER_GUIDE.md` (раздел «Полный отчёт») — там описаны новые поля и рекомендации по заполнению.

## 5. Введение в эксплуатацию

1. После внутренней проверки распространите обновлённый эталонный шаблон среди инженеров.
2. Обновите рабочие инструкции и чек-листы контроля качества, добавив шаги проверки разделов 8, 13, 19 и 20.
3. Запланируйте повторную генерацию ранее выданных отчётов при необходимости согласования с заказчиком.

Следуя этим шагам, вы обеспечите корректный переход на расширенный формат DO_TSS и сохраните совместимость проектов между версиями.
