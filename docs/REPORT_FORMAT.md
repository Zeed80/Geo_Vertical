# Формат профессионального отчета

## Структура PDF отчета

GeoVertical Analyzer генерирует профессиональные PDF отчеты, соответствующие требованиям геодезического контроля.

### Титульный лист

- **Заголовок**: ОТЧЕТ по результатам геодезического контроля антенно-мачтового сооружения
- **Информация об объекте**:
  - Наименование объекта
  - Местоположение  
  - Тип сооружения
  - Дата и время обследования
  - Организация и исполнитель
  - Количество точек и поясов
  - Используемая программа

### 1. Нормативная база

Раздел содержит ссылки на применяемые нормативные документы:

**СП 70.13330.2012**
- Название документа
- Формула допуска вертикальности: **d ≤ 0,001 × h**
- Пояснение параметров

**Инструкция Минсвязи СССР от 23.04.1980**
- Название документа
- Формула допуска прямолинейности: **δ ≤ L / 750**
- Пояснение параметров

### 2. Исходные данные

Раздел содержит:
- Краткое описание объема измерений
- Таблицу координат измеренных точек:
  - № точки
  - X (м)
  - Y (м)
  - Z (м)

**Особенности:**
- Отображаются первые 15 точек
- Если точек больше, добавляется строка "..." 
- В конце указывается общее количество

### 3. Результаты расчетов

Детальный раздел с результатами обработки данных:

**Текстовое описание:**
- Количество выделенных поясов
- Методика обработки данных
- Используемые алгоритмы

**Таблица результатов:**

| № | Выс. (м) | X центра (м) | Y центра (м) | Откл. от верт. (мм) | Допуск (мм) | Статус |
|---|----------|--------------|--------------|---------------------|-------------|---------|
| 1 | 10.0     | 5.123        | 0.045        | 8.5                 | 10.0        | ✓ Норма |
| 2 | 20.0     | 5.145        | 0.089        | 15.2                | 20.0        | ✓ Норма |
| 3 | 30.0     | 5.178        | 0.145        | 25.6                | 30.0        | ✓ Норма |

**Цветовая индикация:**
- Строки в норме - стандартный фон
- Строки с превышением - красный фон
- Статус "✓ Норма" - зеленый текст
- Статус "✗ Превышение" - красный текст

**Статистика:**
- Всего поясов
- В пределах нормы (количество и процент)
- Превышение допуска (количество и процент)

**Детализация превышений** (если есть):
- Номер пояса
- Высота
- Величина превышения в мм

### 4. Графический анализ

**4.1. График вертикальности**
- Высококачественное изображение графика (150 DPI)
- Отклонения от вертикали с цветовой индикацией
- Линии допуска
- Подпись: "Рис. 1. Отклонения центров поясов от вертикальной оси мачты"

**4.2. График прямолинейности**
- Высококачественное изображение графика (150 DPI)
- Стрелы прогиба с линией допуска
- Базовая линия прямолинейности
- Подпись: "Рис. 2. Отклонения от прямолинейности (стрела прогиба)"

### 5. Выводы и рекомендации

Финальный раздел с заключением по результатам контроля.

**Если все в норме:**
1. Констатация соответствия нормативам
2. Указание максимального отклонения
3. Оценка технического состояния: **УДОВЛЕТВОРИТЕЛЬНОЕ**
4. Рекомендации по продолжению эксплуатации

**Если есть превышения:**
1. Указание количества превышений
2. Максимальное превышение
3. Оценка технического состояния: **ТРЕБУЮЩЕЕ ВНИМАНИЯ**
4. Детальные рекомендации:
   - Детальное обследование
   - Корректировка геометрии
   - Повторные измерения
   - Усиленный контроль

- Текст справа: "Дата: ДД.ММ.ГГГГ"

- **Заголовки разделов**: Helvetica-Bold, 12pt
- **Основной текст**: Helvetica, 10pt
- **Таблицы**: Helvetica, 8-10pt
- **Подписи к рисункам**: Helvetica, 9pt, курсив

### Цветовая схема

- **Заголовки таблиц**: #34495e (темно-синий)

- **Интервал между абзацами**: 6 пунктов

## Полный отчет по форме ДО ТСС

GeoVertical Analyzer теперь формирует полностью структурированный документ «Технический отчет по результатам диагностического обследования» (DO_TSS_2). DOCX включает:

- титульные развороты с карточкой объекта (наименование, инв. №, оператор, местоположение, город, год);
- сведения о заказчике/исполнителе, специалистах, приборах и документах (таблицы соответствуют табл. 1–4 оригинального отчёта);
- нагрузки, воздействия и климатические параметры (снег, ветер, гололёд, сейсмичность, коэффициент ответственности, холодный/тёплый периоды);
- описание конструкций и таблицу элементов решётки/поясов;
- полный набор протоколов: журнал угловых измерений, таблицы отклонений, стрелы прогиба, толщинометрия металла, ЛКП, УЗК, прочность бетона, защитный слой, вибрации, осадки фундаментов и расчёт остаточного ресурса;
- раздел с рекомендациями, приложениями и реестром приложений A–M.

Обновлённый билдер также автоматически формирует:

- **Раздел 8. Перечень объектов** — таблицу с наименованием, инвентарным номером, годом ввода и примечаниями.
- **Раздел 13. Результаты анализа технической документации** — консолидированный вывод по каждому документу.
- **Раздел 19. Оценка технического состояния** — классификацию конструкций с комментариями.
- **Раздел 20. Выводы** — нумерованный список ключевых итогов, синхронизированный с расчётом остаточного ресурса и геодезическими измерениями.

Во вкладке `📘 Полный отчет` пользователь заполняет все данные структуры DO_TSS_2. Все таблицы поддерживают добавление/удаление строк, ввод диапазонов через `/`, а состояние формы автоматически сериализуется в проект `.gvproj`.

### Шаблоны

- Шаблоны хранятся в каталоге `user_data/report_templates`.
- Для редактирования и создания шаблонов используйте диалог «Шаблоны полного отчета» (кнопка ✏️ в правой панели виджета отчета).
- Диалог позволяет:
  - загрузить/удалить или переименовать шаблон;
  - отредактировать JSON-описатель (включая новые блоки `title_object`, `angle_measurements`, `annexes`, `reference_profiles`);
  - сгенерировать типовой черновик на основе `docs/DO_TSS.md` / `docs/DO_TSS_2.md`;
  - сохранить шаблон под произвольным именем и обновить общий справочник «ФИО → приборы/сертификаты» (`user_data/report_templates/_reference_directory.json`).

### Автоматическое заполнение

При генерации полного отчета приложение:

1. Загружает выбранный шаблон.
2. Автоматически подставляет результаты расчетов (отклонения, стрелы прогиба, сводки толщинометрии, остаточный ресурс).
3. Формирует DOCX с разворотами, таблицами специалистов, перечнями приборов, приложениями и выводами.

### Как сформировать полный отчет

1. Выполните расчет вертикальности/прямолинейности.
2. Перейдите на вкладку `📘 Полный отчет`, заполните поля вручную или загрузите готовый шаблон (кнопка ✏️ открывает редактор и справочник).
3. Нажмите «Автозаполнение», чтобы программа подставила расчётные разделы (вертикальность, прямолинейность, протоколы толщинометрии/ЛКП/УЗК, остаточный ресурс) и данные справочника специалистов по ФИО.
4. Нажмите «Сформировать полный отчёт» и укажите имя файла DOCX — документ будет собран автоматически.

## Процесс генерации отчета

### Шаг 1: Выполнение расчетов
```
Расчет → Рассчитать (F5)
```

### Шаг 2: Сохранение отчета
```
Файл → Сохранить отчет... (Ctrl+S)
или нажать кнопку "💾 Сохранить отчет"
```

### Шаг 3: Выбор формата
```
Выбрать "PDF файлы (*.pdf)"
```

### Шаг 4: Ввод информации

Откроется диалог для ввода данных:

**Информация об объекте:**
- Наименование объекта
- Местоположение
- Тип сооружения

**Организация:**
- Название организации
- ФИО исполнителя
- Должность

**Дата обследования:**
- Выбор даты из календаря

**Примечания:**
- Дополнительная информация
- Погодные условия
- Особенности измерений

### Шаг 5: Генерация

После нажатия "OK" программа:
1. Собирает все данные
2. Генерирует таблицы
3. Экспортирует графики
4. Формирует PDF документ
5. Сохраняет файл

**Время генерации:** 3-10 секунд в зависимости от объема данных

## Формат имени файла

По умолчанию предлагается имя:
```
Отчет по геодезическому контролю.pdf
```

Рекомендуемый формат:
```
Отчет_[Объект]_[Дата].pdf

Примеры:
- Отчет_Мачта_5_31.10.2024.pdf
- Отчет_АМС_Север_31_10_2024.pdf
```

## Требования к PDF

- **Версия**: PDF 1.7
- **Размер страницы**: A4 (210 × 297 мм)
- **Разрешение графиков**: 150 DPI
- **Сжатие**: Умеренное
- **Совместимость**: Все программы чтения PDF

## Типичный размер файла

- **Без графиков**: 50-100 KB
- **С графиками**: 300-500 KB
- **Большие объекты**: до 1 MB

## Печать отчета

Рекомендуемые настройки печати:
- **Ориентация**: Книжная
- **Масштаб**: 100%
- **Поля**: По умолчанию
- **Цвет**: Цветная или ч/б (графики читаемы в обоих вариантах)

## Архивирование

Рекомендуется сохранять отчеты для:
- Сравнения результатов во времени
- Юридической документации
- Технического паспорта объекта
- Планирования ремонтов

## Экспорт в другие форматы

Из PDF отчета можно:
- Извлечь графики (изображения PNG/JPEG)
- Скопировать текст
- Конвертировать в Word (с сохранением форматирования)
- Распечатать на бумаге

## Excel формат (альтернатива)

Для табличных данных доступен формат Excel:
- 3 листа: Исходные данные, Результаты, Нормативы
- Форматированные таблицы
- Без графиков
- Удобен для дальнейшей обработки

## Пример использования

```python
# В программе:
1. Загрузить данные
2. Выполнить расчет
3. Файл → Сохранить отчет
4. Выбрать PDF
5. Заполнить информацию об объекте
6. OK → Файл готов!
```

Результат: Профессиональный отчет готов для предоставления заказчику или архивирования.
