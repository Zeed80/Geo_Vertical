# История изменений GeoVertical Analyzer

## [1.4.0] - 2025-11-25

### ✨ Мастер построения башни
- Переосмыслен процесс создания башни: конструктор встроен напрямую в правую панель 3D‑редактора. Вкладка «Конструктор» появляется только после выбора `Файл → Создать башню...`, затем пользователь задаёт высоту, нажимает «Применить» и сразу получает базовые пояса с live‑обновлением точек.
- Добавлены быстрые шаблоны («Призма 50 м», «Пирамида 80 м», «Мачта 120 м»), автоматически заполняющие параметры башни и таблицу секций.
- Таблица секций теперь работает снизу вверх и показывает сразу две величины: абсолютную отметку пояса и высоту секции. Основание фиксировано на 0.000 м, верхняя секция равна общей высоте башни, промежуточные пояса можно задавать как абсолютными, так и относительными значениями.
- Добавлена кнопка «Разбить на секции»: башня автоматически делится на заданное количество поясов с равным шагом, после чего высоты можно уточнить вручную.
- Логика секций переведена на порядок снизу вверх: первая строка таблицы — основание, размеры пирамиды рассчитываются автоматически с убыванием к вершине.
- Пункт меню `Файл → Создать башню...` (Ctrl+Shift+N) теперь только переводит интерфейс в режим конструктора и не открывает отдельных диалогов.
- Реализован модуль `core/tower_generator.py`, обеспечивающий генерацию регулярных поясов, крена, девиации точек и точки стояния. Blueprint хранится в проекте и восстанавливается при загрузке *.gvproj*.
- Создан виджет `gui/tower_preview_widget.py` на PyQtGraph GLView для визуального контроля конфигурации прямо внутри мастера.
- Добавлены модульные тесты `tests/test_tower_generator.py`, покрывающие призматические и усечённые башни, изменение радиуса и параметры точки стояния.

### 💾 Сохранение проектов
- `ProjectManager` теперь сериализует состояние мастера (`tower_builder_state`) в файлы проектов и автосохранения. Загруженные проекты автоматически восстанавливают последний blueprint.
- Кнопка автосохранения и команды «Сохранить» сохраняют данные секций и параметры мастера одновременно.

### 📚 Документация
- В `docs/USER_GUIDE.md` добавлен раздел о мастере создания башни: пошаговая инструкция, описание предпросмотра и возможности комбинировать секции разной формы.
- `CHANGELOG.md` дополнен новым релизом 1.4.0.

## [1.3.0] - 2025-11-07

### ✅ Переход на PyQt6
- Переведены все модули GUI на `PyQt6` и `PyQt6-WebEngine`, обновлены импорты, сигналы и обработка диалогов.
- Переписаны обращения к перечислениям Qt на стиль PyQt6 (`Qt.AlignmentFlag`, `Qt.Orientation` и т.д.), слой совместимости более не требуется.
- Обновлены требования (`requirements*.txt`), установщики (`install_windows*.bat`, `install.sh`, `install_windows.ps1`) и инструкции (`README_INSTALL.md`, `INSTALL_WINDOWS.md`, `INSTALL_QUICK.md`).
- В установщики добавлена установка `pytest`, чтобы тесты можно было запускать сразу после настройки окружения.
- Обновлён Windows-установщик: автоматическая установка GDAL сначала пробует PyPI, затем зеркало `download.lfd.uci.edu`, при неудаче выдаёт чёткие инструкции.
- Обновлён backend Matplotlib на `backend_qtagg`, PyQtGraph работает в режиме PyQt6.

### 💅 Улучшения интерфейса
- Добавлена тёмная тема с переключателем в меню `Вид`, тема сохраняется в `QSettings` и применяется ко всему приложению.
- Улучшена поддержка HiDPI: включена политика `HighDpiScaleFactorRoundingPolicy.PassThrough` и использование пикселей высокого разрешения по умолчанию.
- Перепроектированы кнопки тулбаров и диалогов под новые размеры (компактные макеты), обеспечена адаптация под скейлинг.

### 🧪 Тестирование
- Запуск `pytest` подтверждает корректность модульных тестов после миграции.

#### Синхронизация версий
- Обновлена версия в `core/__init__.py` с 1.0.0 до 1.2.0
- Синхронизирована версия в `PROJECT_SUMMARY.md`
- Единая версия 1.2.0 во всём проекте

### ✨ Новые возможности

#### Система исключений (`core/exceptions.py`)
- Добавлена иерархия кастомных исключений
- 15 типов специализированных исключений
- Поддержка контекстной информации в исключениях

Основные исключения:
- `GeoVerticalError` - базовое исключение
- `DataLoadError` - ошибки загрузки данных
- `CalculationError` - ошибки расчетов
- `NormativeError` - ошибки проверки нормативов
- `ReportGenerationError` - ошибки генерации отчетов
- `FilteringError` - ошибки фильтрации точек
- `CoordinateSystemError` - ошибки работы с СК
- `TrimbleError` - ошибки работы с Trimble форматами

#### Система логирования (`utils/logging_config.py`)
- Централизованное логирование
- Логирование в консоль и файлы
- Автоматический rotation (10 MB для основного, 5 MB для ошибок)
- Отдельный файл для ошибок (`logs/errors.log`)
- Настраиваемые уровни (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- Интеграция с `main.py`

#### Улучшения `main.py`
- Добавлено логирование запуска приложения
- Обработка критических ошибок с логированием
- Вывод версии при запуске
- Детальное логирование этапов инициализации
